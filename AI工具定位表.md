# AI 工具定位表

> 核心原则：知道"什么工具解决什么问题"，不盲目追工具。

## 我的主力工具

| 工具名称 | 主要用途 | 适合任务 | 我现在是否需要 | 我准备如何使用 | 可能的问题 |
|----------|----------|----------|:------------:|----------------|------------|
| **豆包** | 日常学习辅助、快速问答 | 概念解释、翻译、笔记整理、碎片化学习 | ✅ 需要 | 日常学习中遇到不懂的概念随时问，替代搜索引擎做知识速查 | 深度有限，不适合复杂推理；英文语境下偶尔不够精准 |
| **DeepSeek** | 通用万能工具，深度推理 | 复杂问题拆解、代码理解、长篇写作、逻辑推理、数学 | ✅ 需要 | 作为主力推理引擎，处理豆包搞不定的深度任务；写代码、分析问题、读论文 | 高峰期可能限流；上下文过长时偶尔遗漏细节 |
| **Claude Code** | AI 编程与 Agent 工作流 | 项目开发、代码重构、多文件编辑、Agent 编排、动态工作流 | ✅ 需要 | 用于学习 AI 编程、理解 Agent 工作流机制、搭建个人项目 | 需理解基本代码逻辑和终端操作；token 消耗大，需注意用量；中文场景不如国产模型自然 |

## 准备探索的工具

| 工具名称 | 主要用途 | 适合任务 | 我现在是否需要 | 我准备如何使用 | 可能的问题 |
|----------|----------|----------|:------------:|----------------|------------|
| **Gemini** | 多模态分析、长文本理解 | 图片识别、视频理解、超长文档分析（1M+ 上下文）、资料检索 | 🔍 可以尝试 | 处理带图片/截图的学习材料；用它做长文论文的快速摘要 | 国内访问不便；中文输出质量不如国产模型；免费版有配额限制 |
| **GitHub Copilot** | IDE 内联代码补全 | 单行/单函数补全、模板代码生成、重复代码消除 | ⏸️ 暂时观望 | 将来在 VS Code 中配合 Claude Code 使用，日常编码提速 | 补全质量不稳定；可能与 Claude Code 功能重叠；月费 $10 |
| **NotebookLM** | 个人知识库 + AI 分析 | 多文档交叉分析、学习笔记整理、自动生成摘要和播客 | 🔍 可以尝试 | 把学习材料打包进去，让它生成结构化笔记和音频播客，通勤时听 | 目前仅英文 Google 账号可用；中文材料支持待验证 |

## 工具组合策略

```
日常学习 ──→ 豆包（快速、轻量、中文友好）
              ↓ 搞不定
深度任务 ──→ DeepSeek（推理强、通用、免费）
              ↓ 涉及项目开发
代码项目 ──→ Claude Code（Agent 编排、多文件协作）
```

三层递进，按任务复杂度选工具，不浪费 token，不降低效率。

---

## 反思

1. **豆包 + DeepSeek + Claude Code 已覆盖我的大多数场景**，短期不需要贪多
2. **Gemini 的 1M 上下文**是差异化能力，适合未来处理超长文献时激活
3. **NotebookLM 的"播客生成"**是独特功能，适合碎片时间学习，值得尝试
4. **Claude Code 是当前学习重点**——不是因为它是"最好的工具"，而是因为它帮我理解 AI Agent 的工作方式本身，这本身就是学习目标
