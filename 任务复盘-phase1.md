# 任务复盘 — Phase 1

---

## 1. 完成这个任务后，我对 AI 的理解发生了什么变化？

**之前：AI 是一个"更强的搜索 + 写作助手"。**

我用豆包查概念、用 DeepSeek 解释问题，本质上把 AI 当成一个不会烦的老师和不会累的搜索引擎。

**之后：AI 是一套"能力组合"，关键不是模型有多强，而是你怎么编排它们。**

整理 AI 信息那 5 条时，最触动我的不是 GPT-5.6 的跑分，而是这段话：

> Claude Code 动态工作流：AI 自己写编排脚本，驱动数十到数百个并行子 Agent。将 Bun 从 Zig 移植到 Rust——75 万行代码，11 天完成。

这意味着 **AI 已经开始从"回答你的问题"进化到"替你干活"**。我之前对 AI 的理解停留在一对一对话模式，但真正的变化发生在 Agent 编排这一层——让 AI 像团队一样协作。这完全改变了我对"学 AI 到底在学什么"的认知：

- 不是在学某个具体工具的操作
- 而是在学**如何设计 AI 之间的协作关系**

**另一个变化：我开始关注"成本"这个维度。**

之前用免费工具，从不考虑 token 消耗。整理价格战那部分时意识到：当 Agent 工作流跑起来，一次任务可能消耗几十万 token，成本就是真实问题。Uber 四个月烧完全年 AI 预算不是段子。这意味着做 Agent 应用的人，必须同时懂模型能力和成本控制，否则产品跑不通。

## 2. 我是否发现自己之前存在工具焦虑、信息焦虑或伪需求问题？

**有，而且很具体。**

**工具焦虑：**
整理工具定位表之前，我潜意识里觉得"别人在用 Gemini、Copilot、NotebookLM，我是不是也该用？" 但填表时被迫问自己"我现在是否需要"这一列，发现 Gemini 和 NotebookLM 都是"可以尝试"而不是"需要"——我其实没有 1M token 的文档要处理，也没有英文播客的刚需。**焦虑的来源不是缺少工具，而是不清楚自己当前的任务需要什么工具。**

**信息焦虑：**
整理那 5 条 AI 信息时，搜索返回了几十条结果，每条看起来都很重要。但筛选到只剩 5 条后，发现真正对我现阶段有意义的只有两条：价格战（直接影响工具选择成本）和 Claude Code 动态工作流（直接影响我当前学习方向）。**剩下 3 条是"应该知道"但不是"现在需要行动"的信息。** 之前刷 AI 资讯的习惯，大部分时间是"知道了很多，但什么都没变"。

**伪需求：**
写工具定位表时差点把 GitHub Copilot 标为"需要"，但冷静下来想：我现在的代码量根本不需要 IDE 内联补全，Claude Code 已经覆盖了我的代码需求。**想用 Copilot 的真实心理是"别人都在用"，不是"我遇到了它解决的问题"。**

## 3. 我现在最应该优先学习哪 1-2 个 AI 工具？

**第一位：Claude Code（继续深入）**

不是学它的快捷键，而是学三件事：
- **动态工作流脚本怎么写**——这是 Agent 编排的实操入口
- **什么时候用 Agent、什么时候不用**——判断力，不是每个任务都需要多 Agent
- **如何控制 token 消耗**——成本意识

理由：它同时是我当前的学习工具和未来的工作方式。深度掌握一个，胜过浅尝十个。

**第二位：一个国产 Agent 框架（SemaClaw 或 Dify）**

理由：
- 中文友好，部署门槛低
- 有可视化界面，能看到 Agent 工作流的实际运行
- Claude Code 是 CLI 的，搭配一个 GUI 的 Agent 框架，理解会更立体
- 将来做项目，国内部署绕不开国产方案

## 4. 我目前最薄弱的壁垒是什么？

**不是技术，是"独立完成一个项目"的经验。**

具体表现：
- 我可以用 Claude Code 完成任务，但如果没有 AI 引导，我能不能独立规划一个项目的结构？
- 我能看懂 AI 生成的代码，但如果出了问题，我能不能自己找到原因？
- 我整理了 5 条 AI 信息，但让我自己发现一个有价值的 AI 应用方向，我还做不到。

**翻译成一句话：我的"AI 辅助下的执行力"在提升，但"脱离 AI 后的判断力"还没跟上。**

这是最深层的薄弱点。因为如果有一天 AI 工具变得更强，所有人都能"用嘴编程"时，**区别就不是谁更会用 AI，而是谁更知道该让 AI 做什么。**

## 5. 下一步我准备如何通过任务二进入真实场景应用？

**不做练习题，做一个自己真的想用的东西。**

具体想法（待任务二进一步明确）：
- 找一个我日常学习中的真实痛点（比如：豆包/DeepSeek/Claude Code 三个工具的切换太麻烦，能不能做一个工具把它们的回答并排对比？）
- 用 Claude Code 从零搭建
- 必须上线（有链接可访问），拒绝"代码写完了但跑不起来"

**检验标准：任务二结束时，我能不能给另一个人一个链接，让他用起来。**

如果没有真实用户愿意用，那说明我所谓的"项目"只是一个练习作业——这就是 Phase 1 告诉我的最大教训：**产出 = 可被他人使用的成果，不是自己觉得学到了。**
